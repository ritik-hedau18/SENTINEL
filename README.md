# 🚨 SENTINEL: AI-Powered Log Anomaly Detection & Incident Response Platform

SENTINEL is a high-performance, real-time log ingestion, pattern embedding, anomaly detection, and incident alerting platform. It provides a visual dashboard for DevOps and SREs to monitor system health, explore log streams, define threshold rules, and triage AI-highlighted anomalies.

---

## 🛠️ Tech Stack

- **Backend**: Spring Boot 3 + Java 17 + Spring Security (JWT) + Hibernate + PostgreSQL + Redis + Spring Kafka (KRaft mode) + Spring AI
- **Frontend**: React 18 + TypeScript + Vite + Tailwind CSS
- **DevOps**: Docker Compose (PostgreSQL, Redis, Kafka, Backend, Frontend)

---

## 🌟 Key Features

1. **Real-time Log Ingestion**: High-throughput REST ingestion endpoints designed to receive log streams.
2. **AI Anomaly Detection**: Evaluation of logs for outlier pattern recognition, matching stack traces, and flagging high-anomaly scores.
3. **Dynamic Alerting Engine**: Threshold-based alert rule matching (e.g. error rate in window minutes) with active state management.
4. **Incident Response Triaging**: Grouping alerts into triage-able incidents with priorities and states (OPEN, ACKNOWLEDGED, RESOLVED).
5. **Interactive UI Explorer**: Advanced log filtering, real-time metrics charts, and alert status tracking.

---

## ⚙️ Setup and Installation

### Run via Docker Compose (Single Command)
```bash
docker compose up -d --build
```
The application will be accessible at:
- **Frontend**: `http://localhost:8082`
- **Backend API**: `http://localhost:8081`
